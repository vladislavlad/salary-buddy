# AGENTS.md — Salary Buddy

## Обзор проекта

Salary Buddy — SPA для расчёта дат и сумм получения зарплаты в РФ с учётом прогрессивной шкалы НДФЛ, производственного календаря и отпусков.

## Стек

- React 19 + TypeScript + Vite + pnpm
- TanStack Query (управление состоянием запросов)
- shadcn/ui + Tailwind CSS (UI)
- date-fns (работы с датами)
- zod (валидация)
- localStorage (персистентность данных)
- React Context + Provider (state management)

## State management

React Context + Provider, тонкие хуки-обёртки над контекстами:
- `useSalaryProvider` — настройки зарплаты и история изменений оклада (`salaryChanges[]`)
- `useBonusesProvider` — список премий
- `useVacationsProvider` — список отпусков
- `usePaymentsForYear`, `useCalendarForYear`, `usePaymentsStore` — расчёт выплат за год
- `useFactsProvider` — пользовательские правки фактического gross

## Ключевые правила расчёта

### НДФЛ 2026 (прогрессивная шкала РФ)

| Накопленный доход | Ставка |
|---|---|
| До 2 400 000 ₽ | 13% |
| 2.4 — 5 млн ₽ | 15% |
| 5 — 20 млн ₽ | 18% |
| 20 — 50 млн ₽ | 20% |
| Свыше 50 млн ₽ | 22% |

Ставка применяется только к сумме превышения порога. НДФЛ считается от накопленного дохода с начала года, затем вычитается уже уплаченный налог. Реализация в `lib/ndfl.ts`.

### Производственный календарь

- Данные загружаются из `https://isdayoff.ru/api/getdata?year=YYYY&cc=ru&holiday=${SEPARATE_HOLIDAY_AND_WEEKEND}`
- Параметр `holiday=1` (константа `SEPARATE_HOLIDAY_AND_WEEKEND`) разделяет праздники (код 8) и выходные (код 1)
- API возвращает строку цифр: `0`=рабочий, `1`=нерабочий, `2`=сокращённый, `8`=праздник
- Если дата выплаты попадает на выходной (суббота/воскресенье) или праздник — смещаем **назад** на ближайший рабочий день (`findPreviousWorkday`)
- Выходные и праздники на календаре: красный цвет

### Изменения оклада

Модель `SalarySettings` содержит массив `salaryChanges[]`, где каждая запись — `{ id, effectiveDate, amount }`. Для любой даты берётся оклад из записи с ближайшей `effectiveDate` не позднее этой даты. Пользователь может добавлять несколько записей через `SalaryManager`.

### Распределение зарплаты

Два режима:
1. **50/50**: аванс = 50% оклада, зарплата = 50% оклада
2. **По отработанным дням**: аванс пропорционален рабочим дням с 1 по 15, остаток — в зарплате

Аванс платится на `advancePaymentDay` месяца M, зарплата — на `salaryPaymentDay` месяца M+1. В декабре — зарплата за 1 рабочий день до последнего рабочего дня месяца.

### Премии

- Пользователь добавляет премию с датой и суммой
- Типы: «В окладах» (`salaries`) — gross = amount × текущий оклад; «Своя сумма» (`custom`) — фиксированная gross-сумма (пользователь вводит сумму до НДФЛ)
- Премия учитывается в накопленном доходе, влияя на ставку НДФЛ для последующих выплат

### Отпуска

- Пользователь добавляет отпуска с датой начала, количеством календарных дней и типом («Оплачиваемый» / «За свой счёт`)
- Тип `Vacation`: `startDate` + `calendarDays` + `dates[]`, без `endDate`
- Массив `dates[]` вычисляется через `computeVacationDates()` — исключает официальные праздники (по ТК РФ отпуск автоматически продлевается)
- Можно добавлять несколько отпусков за год
- **Расчёт отпускных** (по ТК РФ): `СДЗ = доход12мес / 12 / 29,3`, `Отпускные = СДЗ × календарных дней`
- Выплата отпускных — не позднее чем за 3 календарных дня до начала отпуска, с `findPreviousWorkday` (тип `'vacation'`, фиолетовый на календаре)
- Отпускные облагаются НДФЛ по прогрессивной шкале и влияют на накопленный доход
- **Вычитание из ЗП/аванса**: рабочие дни отпуска вычитаются из соответствующих выплат (`dayRate = оклад / рабочиеДниМесяца`)
  - По отработанным дням: аванс считается только от рабочих дней без отпуска (1–15 минус отпускные)
  - 50/50: вычитаем из аванса; если ушёл в минус — переносим избыток на зарплату

### Facts (правки gross)

- `FactsProvider` хранит маппинг `{ paymentId: factGross }` в localStorage
- При расчёте НДФЛ вместо расчётного gross используется `facts?.[event.id]`, если задано
- Позволяет пользователю скорректировать фактическую сумму выплаты

## Цветовая схема календаря

- Зелёный — день выплаты зарплаты (аванс/зарплата)
- Фиолетовый — день выплаты отпускных
- Синий — день выплаты премии
- Серый — дни отпуска (скругление только по краям серии)
- Красный — выходной или праздник

## Конвенции кода

- **Строгий TypeScript**: `strict: true`, без `any`
- **Именованные экспорты** по умолчанию, дефолтный экспорт только для компонентов страниц
- **camelCase** для переменных/функций, **PascalCase** для компонентов и типов
- **zod** для валидации пользовательского ввода и данных из localStorage
- **TanStack Query** для всех асинхронных запросов (загрузка календаря)
- Комментарии только на русском языке, если они нужны
